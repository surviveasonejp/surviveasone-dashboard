# SAO – Situation Awareness Observatory

ホルムズ海峡封鎖シナリオで、日本のエネルギー・食料・水道の供給制約の波及を可視化するリスクシナリオ・シミュレーター。

> **これは予測ではなくシミュレーションです。** 国際協調/標準対応/需要超過/停戦・回復の4シナリオで分析し、前提条件・計算式・データソースを全て公開しています。IEA協調備蓄放出・代替供給ルート・需要削減政策等の対応が取られた場合の影響も明示しています。

**Live:** [surviveasonejp.org](https://surviveasonejp.org) | **API:** [surviveasonejp.net/api](https://surviveasonejp.net/api) | **X:** [@surviveasonejp](https://x.com/surviveasonejp)

## 誰のためのツールか

乳幼児・子育て家庭・透析・在宅医療・介護——インフラ停止時に特別な備えが必要な家庭は日本人口の約20%（5人に1人）。

- 乳幼児・要介護高齢者・障害のある家族を持つ世帯
- 在宅人工呼吸器・透析など医療機器に依存する患者の家族
- 自治体防災担当・地域包括支援センター・福祉避難所運営者

「わが家に何が足りないか」を確認するためのツール。買い占めではなく、過不足の確認を。

## シミュレーション前提

### エネルギー依存構造

| 指標 | 値 | 出典 |
|---|---|---|
| 中東石油依存率 | 94% | JETRO / 財務省貿易統計 (2025年) |
| 石油備蓄 | 229日分（国家143日+民間82日+産油国共同4日、放出中・計45日分放出済み） | 経産省 (2026年4月5日時点推計) |
| LNG全量在庫 | 約25日分（ホルムズ直接依存は6.3%） | 経産省ガス事業統計 / JETRO |
| 火力発電比率 | 65%（LNG29.1% + 石炭28.2% + 石油1.4% + その他6.3%） | ISEP 電力調査統計 (2024年暦年速報) |
| 稼働原発 | 15基（関西7基/九州4基/東京1基/四国1基/東北1基/中国1基） | 原子力規制委員会 |

**注意:** 石油備蓄229日分（放出中）はIEA基準で国際的に充実した水準です。放出制約・精製変換効率を考慮した実効値は約120〜160日。4/10 高市首相が追加20日分放出（5月開始）を発表、全計画完了後は合計約177日見込み。LNGのホルムズ直接依存は6.3%ですが、封鎖による保険料高騰・船舶退避は非依存ルートにも波及し得ます。

### 4シナリオ（政策対応軸）

| シナリオ | 石油遮断率 | LNG遮断率 | 需要変動 | 前提 |
|---|---|---|---|---|
| **国際協調**（代替航路・IEA備蓄放出・需要抑制） | 50% | 3% | -15%削減 | 速度制限/在宅勤務等の政策実施 |
| **標準対応**（国家・民間備蓄放出・代替供給限定） | 94% | 6.3% | -5%削減 | 現実的な政策対応の標準想定 |
| **需要超過**（代替ルートも混乱・需要増加） | 100% | 15% | +10%増加 | 供給制約が最大化する参考ケース |
| **停戦・回復**（45日目停戦→段階的正常化） | 8%（最終） | 0.5%（最終） | +8%（停戦後一時反動） | 保険解除14日→港湾再開30日→契約60日→残留リスク8% |

### 計算モデル（16式）

| モデル要素 | 実装内容 |
|---|---|
| **フロー型在庫モデル** | `dStock/dt = Inflow - Consumption + SPR_Release + AlternativeSupply`。365日の日次在庫推移 |
| **LNG供給途絶モデル** | 消費は需要ベース。ホルムズ依存分(6.3%)のみ途絶、非ホルムズ供給(93.7%)は継続 |
| **SPR放出メカニズム** | 国家備蓄: 14日リードタイム + 日次上限30万kL。民間: 即日・実質70% |
| **封鎖解除曲線** | シナリオ別に `blockadeRate(t)` を時間関数化。段階的解除 |
| **需要破壊モデリング（価格媒介型）** | WTI原油価格連動。価格上昇率→需要削減率を弾力性ベースで計算。在庫閾値との複合 |
| **段階的制約閾値** | 50%→価格上昇（需要超過）、30%→供給制限（奇数偶数制）、10%→配給制（政府管理分配）、0%→完全停止 |
| **経済カスケード** | WTI実測値を基準価格に使用。原油→ガソリン(弾力性0.7)→物流(0.3)→食品(0.15)。IEA+1973年石油ショック実績ベース |
| **連系線融通** | OCCTO運用容量ベース、10本の連系線、非対称容量対応。3回反復で多段融通安定化 |
| **原子力補正** | 地域別に稼働原発出力を反映。設備利用率80%。最大70%カバー |
| **再エネバッファ** | 太陽光(CF15%)/風力(CF22%)/水力(CF35%)の地域別設備容量。蓄電池なし上限40% |
| **水道崩壊カスケード** | 電力停止→同日水圧低下→翌日断水→3日後衛生悪化 |
| **廃棄物カスケード** | 石油供給制限+3日→ゴミ収集停止（収集車燃料枯渇）、電力停止→焼却炉停止 |
| **食料サプライチェーン** | ナフサ→石化製品→包装材の連鎖的制約。化学日報報道ベース |
| **物流制約モデル** | 石油在庫連動の物流稼働率(100%→70%→30%→0%)。地域別トラック燃料依存率×配送遅延で制約日算出。地図上に地域間供給フロー可視化 |
| **代替原油精製互換性** | 非中東原油（米国ガルフ/西アフリカ）の精製互換性ペナルティ。API度差・硫黄分差による効率補正 |
| **代替供給ルート** | フジャイラ/ヤンブー/非中東の3ルート。調達成功率は国際競争で日次低下 |

## Features

- **封鎖Day Nカウンター** — 封鎖開始（Day 0 = 2026/04/06 IRGC停止命令）からの経過日数をリアルタイム表示。シナリオ別「解除推定 Day M / あとN日」プログレスバー付き
- **Supply Timeline** — 石油/LNG/電力の供給可能日数カウントダウン（国際協調/標準対応/需要超過/停戦・回復の4シナリオ切替）。回復タイムラインスライダー・代替ルートパネル・政策マイルストーンタイムライン搭載
- **Impact Map** — 全国10電力エリアの供給影響マップ（連系線融通・原子力・再エネ・物流制約補正込み。地域間石油供給フロー可視化。沖縄は先島諸島まで表示。GPS/localStorage/手動の3段階フォールバックでエリア自動選択）
- **Tanker Tracker** — 実在タンカー30隻（VLCC13+LNG14+Chemical1+Suezmax2）の推定航跡マップ。ホルムズ未通過船・日本向けでない船のグレーアウト表示。ETAは経過日数で自動減算。供給元カテゴリ別到着タイムライン（代替ルート amber/米国ガルフ blue/LNG green）+ 備蓄残量カーブ重ね表示。米国産原油タンカー（喜望峰回り）を青色で区別可視化。シナリオ連動代替ルート可視化（TM-A/B/C: 容量比例線幅・供給ギャップチャート）
- **Petrochem Tree** — ナフサ→石化製品→包装材の7段階樹形図。クラッカー収率ゲージ（エチレン~30%/プロピレン~16%/ブタジエン~4%）・代替フィード（石炭MTO/エタン）ノード・供給制約進行によるグレー化アニメーション・川下バッファ（ポリマー在庫105日）表示
- **Food Supply** — 衛生・包装への供給制約が食料より先に進む。ナフサ→石化→包装材の連鎖的制約モデル（商品カテゴリ別在庫日数・備蓄優先順位）
- **Household Supply Check** — 家庭の備蓄量入力 → 供給可能日数の目安を確認。残り14日未満のボトルネック資源について「あとN日で限界 → 今すぐやること3項目」を個別表示。Xシェアは短く個人化（「我が家はあとN日」形式）。計算はブラウザ内で完結、サーバーへの送信なし
- **Prepare** — フェーズ別行動指針（配給制対応含む）+ 備蓄チェックリスト + 行動チェックリスト（情報・医療・移動・住宅・コミュニティ）+ 要配慮者チェックリスト（乳幼児/医療機器/透析/介護/障害）
- **Methodology** — 16式のシミュレーションモデル解説 + 国際対応パターン比較表（IEA/OPEC+/各国政府令）+ データソース一覧 + 感度分析
- **AISタンカー追跡** — AISStream.ioから1日2回、位置・目的港・日本向け判定を自動取得
- **Supply Buffer表示** — 石油備蓄241日分・LNG代替供給継続率・3つの代替ルートを安心情報として冒頭表示
- **ライトモード** — デフォルトはライトモード。ユーザー設定・OS設定に応じて自動切替。公共機関・データポータルとしての信頼性を重視
- **PWA** — オフラインキャッシュ対応（重要APIデータをプリキャッシュ。停電後も参照可能）
- **アクセシビリティ** — ARIA属性・スクリーンリーダー対応
- **FAQ構造化データ** — 「停電 赤ちゃん ミルク」「人工呼吸器 停電 対策」等の検索で強調スニペットを狙うSchema.org FAQPage
- **API Docs** — 23エンドポイントのインタラクティブドキュメント + OpenAPI 3.0 + AI Plugin
- **意思決定支援UI** — 回復タイムラインスライダー（封鎖解除タイムラインのインタラクティブ操作）・代替ルートパネル（ルート別供給余力表示）・政策マイルストーンタイムライン

## タンカー追跡

VLCC 13隻 + LNG 14隻 + Chemical 1隻 + Suezmax 2隻 = 30隻を追跡（日本向けでない船は「日本向けでない」バッジ付きで表示）。全船のIMO番号はMaritimeOptima/VesselFinderで検証済み（2026年4月12日時点）。

**【4/6-7 重大事象】** カタールLNG船 AL DAAYEN（IMO 9325702）・RASHEEDA（IMO 9443413）が戦争開始後初のホルムズ通過を試みるもIRGCが停止命令→引き返し。イランは「仮停戦でも海峡は再開しない」と表明（Bloomberg 4/6）。

- **ホルムズ未通過判定**: ペルシャ湾内出発港（Ras Tanura, Jubail, Kharg Island, Ras Laffan, Mina Al Ahmadi, Basrah）からの船舶はグレーアウト + 「封鎖時到達不可」バッジ
- **引き返し表示**: ホルムズ通過を試みて引き返したカタールLNG船を専用バッジで表示
- **非日本向け判定**: destinationPortが日本港リストにない船舶はグレーアウト + 「日本向けでない」バッジ
- **供給元カテゴリ別タイムライン**: ホルムズ経由（gray）/ 代替ルート中東発（amber）/ 非中東調達・米国ガルフ（blue）/ LNG（green）の4カテゴリで区分表示
- **米国ガルフ可視化**: TATESHINA（Ingleside発・喜望峰回り）を非中東調達として青色バーで表示。備蓄カーブと同軸で「代替調達37日 vs 代替ルート4日」の差を可視化
- **ETA自動減算**: `meta.updatedAt`からの経過日数で`eta_days`を自動補正。デプロイ不要で鮮度維持
- **4/12時点の状態**: MAYASAN/YAKUMOSAN（4/11 東進確認） / 引き返し2隻（カタールLNG AL DAAYEN・RASHEEDA）/ 代替ルートSuezmax2隻追加 / 合計30隻追跡
- **AIS日本向け判定**: 目的港フィールドから`JP`プレフィクス・日本港名24件辞書で自動判定
- **AIS ETA再計算**: 現在位置+SOGから目的港までの大圏距離でリアルタイムETA算出

## Architecture

```
Client (React 19 + Vite 6 + Tailwind CSS 4)
  ├── PWA Service Worker (App Shell + API プリキャッシュ)
  └── fetch /api/*
Dashboard Worker (Cloudflare Workers)
  ├── D1 (SQLite) — reserves/consumption/regions/electricity/oil_prices
  ├── KV — API cache + AIS positions + tanker overrides
  ├── R2 — OWID CSV archive
  └── Cron Triggers (4/5枠)
        ├── 毎週月曜 03:00 UTC — OWID energy-data
        ├── 毎日 06:00 UTC — AIS位置+目的港 (2回目)
        ├── 毎日 18:00 UTC — 電力需給 + AIS (1回目) + WTI原油価格
        └── 毎月18日 06:00 UTC — 石油備蓄 + LNG在庫
Notify Worker (surviveasone-ops)
  ├── Workers AI — 日次サマリのLLM要約（Llama 4 Scout / Gemini切替可）
  ├── KV — 前回シミュレーション状態 + RSS既読管理
  └── Cron Trigger (1枠)
        └── 毎日 09:00 UTC — 差分検知 + 日次サマリ + RSS監視 → Discord Webhook
```
